# PRD - 食譜收藏夾系統

## 1. 專案概述

- **背景與動機**
  現代人常因為不知道吃什麼而煩惱，或是手邊有剩餘食材卻不知道如何應用。此外，網路上食譜散落各處，難以有系統地集中管理與搜尋。為了幫助使用者更輕鬆地尋找料理靈感並妥善管理自己的食譜，因此開發此「食譜收藏夾系統」。

- **目標用戶**
  - **一般人**：希望透過食材組合尋找料理靈感、管理自己喜愛的食譜的普通使用者。
  - **管理員**：負責管理平台內容、維護系統運作的人員。

- **核心價值主張**
  提供一個操作簡便的食譜集中平台，協助使用者從現有食材出發找尋靈感，並輕鬆儲存與管理專屬的食譜清單，減少剩食並提升料理樂趣。

## 2. 功能需求

### 1. 儲存食譜
- **作為** 一般人，**我希望** 可以將喜歡的食譜（包含名稱、食材清單、步驟說明）儲存到系統中，**以便** 日後準備料理時能快速查閱，不需要重新上網尋找。

### 2. 瀏覽、搜尋食譜
- **作為** 一般人，**我希望** 能夠透過關鍵字或分類來瀏覽與搜尋全站食譜，**以便** 快速找到我想做的菜餚，獲得料理靈感。

### 3. 從食材組合搜尋食譜
- **作為** 一般人，**我希望** 能夠輸入我手邊擁有的多樣食材，**以便** 系統能篩選並推薦包含這些食材的合適食譜，幫助我消耗剩餘食材。

### 4. 使用者註冊與登入
- **作為** 一般人，**我希望** 可以註冊專屬帳號並安全登入，**以便** 我能擁有獨立的食譜收藏空間，並跨裝置存取我的個人資料。

### 5. 管理員管理食譜與使用者
- **作為** 管理員，**我希望** 能夠在後台檢視及管理全站的食譜內容與使用者列表，**以便** 移除不當內容、維護系統的安全與內容品質。

## 3. 非功能需求

- **技術限制**
  - 需採用 **Flask** 作為後端框架。
  - 前端樣式與版面渲染需使用 **Jinja2** 模板引擎進行整合。
  - 資料庫需使用 **SQLite** 進行本地資料儲存。

- **效能與安全考量**
  - **安全性**：使用者密碼必須經過安全的雜湊加密（如 bcrypt）才能存入資料庫；表單輸入應防止 SQL Injection 與 XSS 攻擊。
  - **效能**：在「從食材組合搜尋食譜」的功能中，由於比對邏輯較複雜，需確保資料庫結構與索引設計良好，以維持合理的載入與回應時間。

## 4. MVP 範圍

### Must Have (必須有)
- 使用者註冊、登入與登出系統。
- 食譜的 CRUD 功能（新增、瀏覽、編輯、刪除）。
- 支援輸入單一或多項關鍵字的基礎食譜搜尋。
- 支援輸入食材組合找尋對應食譜（核心亮點）。
- 基礎的管理者權限判定，允許管理員刪除違規資料。

### Should Have (應該有)
- 食譜的難易度、製作時間與分類標籤（如：家常、甜點、健身）。
- 公開或私有食譜的權限設定。

### Nice to Have (可以有)
- 支援上傳食譜封面圖片。
- 首頁輪播或熱門食譜推薦區塊。
- 針對食譜的評價與留言系統。
